Le mode "rated"

Ce mode permet d'indiquer au script que les votes seront enregistrés

Il faudra pour cela avant tout importer tous les fichiers disponibles à cette adresse : https://github.com/Hanasagi/transitionJS/tree/Julien/stats

Une fois ça de fait, il faudra ajouter la classe "rated" au body comme ceci : <body class="rated">

Il ne faudra pas non plus oublier de changer le fichier config.php et de rajouter ces lignes dans votre fichier HTML

<script src="https://cdn.jsdelivr.net/npm/chart.js@2.8.0"></script> <script src="https://cdn.jsdelivr.net/npm/axios/dist/axios.min.js"></script>

Explications techniques

Le mode présentateur

Ce mode permettra au présentateur de remettre à 0 les votes ainsi que de voir les résultats des votes

On y accède en entrant ?presenter_mode=true dans l'URL

Bien sûr, ce mode est disponible uniquement si le mode "rated" est activé

Comment ça marche ?

Le script teste l'URL pour voir si ?presenter_mode=true y est renseigné

S'il y est alors il supprimera tous les boutons "submit" et les remplacera par un bouton "Voir les résultats" et un bouton "Recharger les votes"

Désavantage de cette technique


- Peu sécurisé

La sauvegarde des votes en PHP

La table quiz dans la base de données se présente comme ceci :

id_quiz question alreadyVoted

La table answers se présente comme ceci :

id_quiz answer_text count

La mise à jour des votes se fait dans le fichier update_answers.php

Problème rencontré lors du développement

Lorsque que l'on valide une ou plusieurs réponse(s), elles sont stockées dans une variable de session

Avec ce système, lorsque que l'on remettait les votes à 0, la variable de session contenait encore les anciens votes

Donc, des compteur de vote < 0 apparaissaient
